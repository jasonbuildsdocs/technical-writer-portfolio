# Documentation Process Example

This sample outlines a simple documentation workflow that can be used across product teams.

## 1. Collect Requirements
- Review new features  
- Speak with Product and Engineering  
- Identify user pain points  
- Confirm expected outcomes  

## 2. Draft Content
- Create outline  
- Draft user guide  
- Add screenshots or examples  
- Ensure terminology matches product UI  

## 3. Cross-Functional Review
- Product validates accuracy  
- Engineering verifies technical details  
- CX confirms clarity  

## 4. Publish
- Add to knowledge base  
- Tag by category  
- Link to related articles  

## 5. Maintain
- Review quarterly  
- Update after releases  
- Monitor analytics for drop-off or confusion
